[Hindi] Training And Test Data In ML - Machine Learning Tutorials Using Python In Hindi

Training and Test Data in Machine Learning

To be able to train the model, you first have to split the existing data into training and test data. The test data are neither used in training nor for manual optimization of the system, but rather serve to test the finished system against unknown data with which it was not trained. Reserving 10 to 20 percent of the data as test data is a good estimate. It is important to select these at random. 

In order to be able to check during the training and optimization process whether the quality of the results has improved, it makes sense to reserve a further part of the data as validation data. However, this reduces the amount of data that is available for training and may make the training process vulnerable to the random selection of the validation data. 

The aim of the training is to minimize the loss function. This is regarded as an optimization problem and is approached to a better solution in small steps. To do this, data sets run through the neural network and generate a forecast. The deviation from the setpoint is calculated using the loss function and this deviation is then propagated through the network in the opposite direction, from the output back to the input.

The model must be initialized at the beginning of the training. If all weights are zero, the training algorithm will not find any gradients, so it is good practice to initialize the models with small random values. There are also ready-made methods for this in the frameworks according to best practices.

If the trained system does not achieve good results with the validation data, it is called underfitting. The model then does not represent the training data well. This can be due to poor or poorly prepared data, insufficient capacity of the model, or the parameters of the training process.

If the results with the validation data are good but significantly worse with the test data, this is called overfitting. The model is trained to correctly map the training data, but it does not generalize. Possible causes for this are too little or no representative training data, i.e. that the feature space is not sufficiently filled by the training data.

We train our machine learning model

During training, we go through several runs: In the beginning, we pass all training data to the machine, i.e. our system, and it decides almost by chance whether there is an apple or a pear in the picture. The model then checks how often it has made the right decision and looks at the “characteristics” of this data: If the shape is more rounds, it is probably an apple. If the color is red, more often it is an apple. If the color is more brownish, it is more likely a pear. All of these “features” are called “features” in machine learning. And the classification (apple/pear) is called “labels”.

After each training run, the model adjusts its decision-making criteria, and an increasingly more precise model is created through “trial and error”. Over time, i.e. only through a large number of training runs, the model can better differentiate between the “labels” and has thus “learned” to differentiate between apples and pears.

We test our model

After we are satisfied with the result and the system has sorted 99% of the images correctly, for example, our test data is used: This shows whether the model can transfer what has been learned to new, previously unknown data (images). At the end of the test run, the model can be revised again if, for example, errors have crept in.

We use the machine learning model in practice

Based on the training and the test, we have now taught a computer program to recognize apples and pears on pictures. Now it is a matter of using this model in the “real world” and classifying new, previously unknown images.
