Django 3 Blog: Creating APIs for Posting Comments on Blog | Python Django Tutorials In Hindi #92

In the previous tutorial, we created a model to store the comments in the database. So, we're already done with half of the work to post comments on our iCoder blog, and the remaining half is to create  API so that the users can comment on the blog posts. Without wasting time, open the urls.py file of the blog app and type the below code in the URL patterns.

path('postComment', views.postComment, name="postComment")

Let's create the views for the path that we've created in the above code. Open the views.py file of the blog app and type the below code:

from django.shortcuts import redirect
from blog.models import Post, BlogComment
from django.contrib import messages
from django.contrib.auth.models import User

def postComment(request):
    if request.method == "POST":
        comment=request.POST.get('comment')
        user=request.user
        postSno =request.POST.get('postSno')
        post= Post.objects.get(sno=postSno)
        comment=BlogComment(comment= comment, user=user, post=post)
        comment.save()
        messages.success(request, "Your comment has been posted successfully")
        
    return redirect(f"/blog/{post.slug}")

We're using a direct logic to post comments. First of all, we're checking if the Http request method is POST or not. If the request method is POST, then only the comment will be posted. After this, we're storing the user's comment in a variable named comment. Similarly, we're storing the other details in different variables. If everything goes right, we will display a success message, after which the user will be redirected to the blog post. But what if there are already some comments associate with a blog post? The code that we've written above will only help us to post a comment and not to display the already posted comments. To display the already posted comments, type the below code:

def blogPost(request, slug): 
    post=Post.objects.filter(slug=slug).first()
    comments= BlogComment.objects.filter(post=post)
    context={'post':post, 'comments': comments, 'user': request.user}
    return render(request, "blog/blogPost.html", context)

In the above code, we're filtering all the comments associated with a blog post with the help of objects.filter(post=post) and storing them in a comments variable to render the comments to the template with the help of context dictionary. After this, open the blogPost.html file and type the below code after the <h2> Comments </h2> :

 <form action="/blog/postComment" method="post">
      {% csrf_token %}
      <input type="text" name="comment" placeholder="Enter comment here">
      <input type="hidden" name="postSno" value="{{post.sno}}" >
      <input type="submit" value="Submit">
</form>

In the above code, we've created a form to type the submit the comment. Now, restart the development server and head over to any blog post. You will see the following screen:

You can see that an input box and a submit button are displayed on the blog in the above image. Type your comment and click on the submit button, and you will receive a success message. Let's check if the comment is sent to the back-end or not. Head over to the http://127.0.0.1:8000/admin/blog/blogcomment/, and you will notice that a blog comment object is visible. I posted a comment, and in the below image, you can see that the comment is successfully sent to the back-end system.

You can see that the comment is posted in the BlogComment object (7) format. Let's change this format, open the models.py file and type the below code at the end of the BlogComment model:

    def __str__(self):
        return self.comment[0:13] + "..." + "by" + " " + self.user.username

Start the development server again, and you will notice that the format is changed. You can also find the same in the below image:

This tutorial ends here, and in the next tutorial, we will render the blog comment on the template.

Code blog/models.py as described/written in the video :

from django.db import models
from django.contrib.auth.models import User
from django.utils.timezone import now


class Post(models.Model):
    sno=models.AutoField(primary_key=True)
    title=models.CharField(max_length=255)
    author=models.CharField(max_length=14)
    slug=models.CharField(max_length=130)
    timeStamp=models.DateTimeField(blank=True)
    content=models.TextField()


    def __str__(self):
        return self.title + " by " + self.author

class BlogComment(models.Model):
    sno= models.AutoField(primary_key=True)
    comment=models.TextField()
    user=models.ForeignKey(User, on_delete=models.CASCADE)
    post=models.ForeignKey(Post, on_delete=models.CASCADE)
    parent=models.ForeignKey('self',on_delete=models.CASCADE, null=True )
    timestamp= models.DateTimeField(default=now)

    def __str__(self):
        return self.comment[0:13] + "..." + "by" + " " + self.user.username
